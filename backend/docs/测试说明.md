# 测试说明

## 添加日程

- “帮我明天下午三点加一个项目会议。”
- “明天开会。”
- “十分钟后提醒我喝水。”
- “每周三下午四点提醒我去实验室。”

## 冲突检测

- 先添加明天下午三点会议
- 再添加明天下午三点和导师沟通
- 观察是否提示冲突

## 重复检测

- 连续两次添加同一时间同一标题的日程
- 观察是否提示疑似重复

## 删除歧义

- 创建多个明天会议
- 说“删除明天的会议”
- 观察是否让用户选择

## 修改歧义

- 创建多个会议
- 说“把明天的会议改到四点”
- 观察是否让用户选择

## 查询总结

- “我明天有什么安排？”
- “我下午三点有空吗？”
- “我这周有哪些会议？”

## 多轮确认

- 用户：“帮我明天下午三点加个会议。”
- AI：“这个时间已有安排，是否换时间？”
- 用户：“那改到四点。”
- AI 继续完成添加流程

## 取消操作

- 用户：“帮我明天下午三点加个会议。”
- AI：“发现冲突，是否仍然添加？”
- 用户：“取消吧。”
- AI：“好的，已取消本次添加。”
