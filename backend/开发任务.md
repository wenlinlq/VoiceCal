可以，用 **Claude Code + DeepSeek V4** 开发这个项目，推荐你按下面这个方式做。

核心思路是：

```text
Claude Code = 你的本地 AI 编程助手，负责读项目、改代码、跑命令、写测试
DeepSeek V4 = 你的推理/编码模型，负责低成本生成和修改代码
你 = 产品经理 + 架构师 + 验收人
```

Claude Code 官方定位就是终端里的代码助手，可以编辑文件、运行命令、管理项目；安装方式包括官方脚本、Homebrew、WinGet 等。([Claude Code][1]) DeepSeek API 支持 OpenAI / Anthropic 兼容格式，最新版 V4 Preview 文档说明可直接把模型名换成 `deepseek-v4-pro` 或 `deepseek-v4-flash`，并继续使用兼容 API。([DeepSeek API Docs][2])

```text

```

curl -fsSL https://claude.ai/install.sh | bash

官方文档说明 Claude Code 是一个 full-featured CLI，可以在终端里编辑文件、运行命令和管理整个项目。([Claude Code][1])

# 三、配置 DeepSeek V4

DeepSeek API 是兼容 OpenAI / Anthropic 风格的，所以你可以把它作为模型供应商接入到你的开发和项目里。DeepSeek V4 Preview 文档里提到模型名可以使用：

```text
deepseek-v4-pro
deepseek-v4-flash
```

一般建议：

```text
deepseek-v4-flash：日常开发、改代码、写测试，便宜快
deepseek-v4-pro：复杂架构、疑难 bug、重要重构
```

环境变量可以这样放到 `.env`：

```bash
DEEPSEEK_API_KEY=你的_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

项目里用 OpenAI 兼容 SDK 调用时，大概是这种形式：

```python
from openai import OpenAI
import os

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
)

response = client.chat.completions.create(
    model=os.getenv("DEEPSEEK_MODEL", "deepseek-v4-flash"),
    messages=[
        {"role": "system", "content": "你是声历 Agent 的 Calendar Agent。"},
        {"role": "user", "content": "明天下午三点提醒我开组会"},
    ],
)
```

# 四、Claude Code 怎么用在这个项目里

你不要一次性让 Claude Code 写完整项目。正确方式是 **小步迭代**。

## 第一步：让它读开发文档

把我刚才给你的 `声历Agent_开发文档.md` 放到项目根目录，然后创建一个 `CLAUDE.md`。

`CLAUDE.md` 可以这样写：

```md
# 声历 Agent 项目开发说明

## 项目目标

本项目是一个基于 AgentScope Runtime 的流式语音日历管理工具。

核心链路：

用户语音 → WebSocket → ASR → Calendar Agent → Tool Calling → Calendar Service → TTS → 前端播放

## 初期开发目标

只做 MVP，不做复杂业务。

优先实现：

1. FastAPI 项目骨架
2. 一个核心 WebSocket 接口 `/ws/voice`
3. 文本调试接口 `/api/agent/text`
4. 本地日历 CRUD
5. Calendar Agent
6. Calendar Tools
7. 后续接入 ASR / TTS

## 技术栈

- Python 3.11
- FastAPI
- WebSocket
- AgentScope Runtime
- DeepSeek V4
- PostgreSQL
- Redis
- Docker Compose

## 代码要求

- 使用异步 FastAPI
- 使用 Pydantic 定义 Schema
- 所有接口返回统一格式
- 工具调用输入输出必须是 JSON
- 初期使用默认用户 `demo_user`
- 先跑通文本链路，再接语音链路
- 每完成一个模块都要写最小测试
```

然后在 Claude Code 里说：

```text
请阅读 CLAUDE.md 和 声历Agent_开发文档.md，理解项目目标。先不要写代码，先输出你理解的项目结构和第一阶段任务清单。
```

这样可以先校准 Claude Code 的理解。

# 五、推荐开发步骤

## 阶段 1：初始化项目

让 Claude Code 执行：

```text
请创建 FastAPI 项目骨架，包含 app/main.py、api、services、agents、tools、schemas、models、core、db 等目录。先实现 GET /api/health，并创建 requirements.txt、.env.example、Dockerfile、docker-compose.yml。
```

你验收：

```bash
docker compose up -d
curl http://localhost:8000/api/health
```

期望返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "running"
  }
}
```

---

## 阶段 2：做本地日历 CRUD

继续让 Claude Code 做：

```text
基于当前项目结构，实现 Event 数据模型、Pydantic Schema 和 CalendarService。初期不接第三方日历，只使用 PostgreSQL 存储 events 表。实现 create_event、query_events、update_event、delete_event。
```

再让它补测试：

```text
请为 CalendarService 编写 pytest 测试，覆盖创建、查询、修改、删除日程。
```

---

## 阶段 3：封装 Calendar Tools

让 Claude Code 做：

```text
请基于 CalendarService 封装 Calendar Agent 可调用的工具函数，包括 add_event_tool、query_event_tool、update_event_tool、delete_event_tool。每个工具输入输出都使用 JSON dict，不要让 Agent 直接操作数据库。
```

这一步很关键，因为你的系统核心是：

```text
Agent 不直接改数据库
Agent 只调用工具
工具再调用 CalendarService
```

---

## 阶段 4：接 DeepSeek V4 做文本 Agent

让 Claude Code 做：

```text
请实现 CalendarAgent，使用 OpenAI 兼容 SDK 调用 DeepSeek V4。读取环境变量 DEEPSEEK_API_KEY、DEEPSEEK_BASE_URL、DEEPSEEK_MODEL。先实现文本链路：用户输入一句中文日历指令，Agent 输出结构化 intent 和 slots，并根据 intent 调用对应 Calendar Tool。
```

然后加一个调试接口：

```text
请实现 POST /api/agent/text，用于文本调试 Calendar Agent。请求包含 session_id 和 text，响应包含 intent、slots、tool_result、reply_text。
```

你用这个测试：

```bash
curl -X POST http://localhost:8000/api/agent/text \
  -H "Content-Type: application/json" \
  -d '{"session_id":"sess_demo","text":"明天下午三点提醒我开组会"}'
```

---

## 阶段 5：实现一个 WebSocket 主接口

让 Claude Code 做：

```text
请实现核心 WebSocket 接口 /ws/voice。初期先不接真实 ASR/TTS，用 text.message 事件模拟语音识别结果。客户端发送 text.message 后，后端调用 CalendarAgent，并返回 agent.reply 和 turn.done。
```

初期事件可以简化：

前端发：

```json
{
  "type": "text.message",
  "session_id": "sess_demo",
  "text": "明天下午三点提醒我开组会"
}
```

后端回：

```json
{
  "type": "agent.reply",
  "text": "好的，已为你添加明天下午三点的组会提醒。"
}
```

```json
{
  "type": "turn.done",
  "success": true
}
```

这个阶段先不碰音频，先把 WebSocket + Agent + Tool 跑通。

---

## 阶段 6：接入 ASR / TTS

等文本链路稳定后，再让 Claude Code 做：

```text
请在 /ws/voice 中增加 audio.chunk 和 audio.end 事件处理，封装 ASRService。ASRService 先定义统一接口 transcribe_stream，具体实现预留给 AgentScope Runtime ASR Client。
```

然后做 TTS：

```text
请封装 TTSService，提供 synthesize_stream(text) 方法。CalendarAgent 生成 reply_text 后，通过 TTSService 返回 tts.chunk 事件。
```

初期可以先 fake：

```text
ASRService 先返回固定文本
TTSService 先返回空音频或 mock base64
```

最后再替换成 AgentScope Runtime 的真实 ASR/TTS。

# 六、你应该怎么给 Claude Code 下指令

不要说：

```text
帮我开发完整语音日历项目
```

这样会乱。

要说：

```text
只完成第一阶段：FastAPI 项目骨架和健康检查接口。不要实现 Agent，不要实现语音，不要实现数据库业务。完成后运行测试并告诉我启动命令。
```

或者：

```text
只实现 CalendarService，不要改动 WebSocket，不要接 ASR/TTS。请先列出将要修改的文件，再执行。
```

或者：

```text
现在修复 pytest 报错。不要重构项目结构，只修改导致测试失败的最小代码。
```

这才是 Claude Code 的正确用法。

# 九、最推荐你的开发路线

你可以直接照这个走：

```text
第 1 步：Claude Code 搭 FastAPI 项目骨架
第 2 步：实现本地 CalendarService
第 3 步：实现 Calendar Tools
第 4 步：接 DeepSeek V4，跑通文本 Agent
第 5 步：实现 /ws/voice，但先用 text.message 模拟语音
第 6 步：接 AgentScope Runtime ASR
第 7 步：接 AgentScope Runtime TTS
第 8 步：前端录音和播放联调
第 9 步：补 README、接口文档、Demo 演示
```

最重要的一句话：

```text
先别一上来做真实流式语音，先用 Claude Code 把“文本 Agent → Tool Calling → 日历操作”跑通。
```

这个跑通以后，ASR/TTS 只是把输入输出换成语音流。
