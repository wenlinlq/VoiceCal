# 声历 VoiceCal — AI 语音日程管家

> 用说话管理一切日程 · 微信小程序 + H5 双端

---

## 一、产品定位

声历是一款基于大语言模型的智能语音日程管理工具。用户通过自然语言与 AI 对话，即可完成日程的创建、查询、修改、删除，并获得定时微信推送提醒。

**一句话：对着手机说话就能管日程。**

---

## 二、核心功能

| 功能 | 说明 |
|------|------|
| 🎙️ 语音交互 | 说话→ASR转录→Agent理解→执行→TTS朗读回复，全程免打字 |
| 📅 日程 CRUD | 自然语言创建/查询/修改/删除日程 |
| 🔔 订阅推送 | 微信服务通知定时提醒，支持 emoji 标题 |
| 🧠 长期记忆 | 记住用户习惯（常去地点、默认时长等），越用越懂 |
| ⚡ Skill Router | 高频操作走快速通道（1次LLM），复杂操作走 Agent 推理 |
| 📱 跨平台 | vue/uni-app 一套代码，编译产出微信小程序 + H5 |

## 四、技术亮点

### 1. Skill Router 分流架构
- 常见操作（增删查）**1 次 LLM 调用**直接执行
- 复杂操作降级 ReActAgent 多轮推理
- 比纯 Agent 循环快 2-3 倍

### 2. 实时 TTS + 文字清洗
- 边生成边合成，首音延迟低
- `_CleanTTSWrapper` 自动过滤 emoji/Markdown
- 朗读内容干净，只听最终回复

### 3. 智能时间解析
- LLM 直接输出 ISO 8601 格式
- 支持「明天下午三点」「下周二」「6月2号」「大后天」「半小时后」等
- 无需手写正则，扩展性极强

### 4. 长期记忆
- Mem0 + Milvus 向量数据库
- Agent 自动记录和检索用户习惯
- 跨会话持久化，越用越懂

### 5. Session 隔离
- Redis 按 `(user_id, session_id)` 命名空间隔离
- 多用户独立短期记忆 TTL、对话上下文、WebSocket 连接
- 降级模式：Redis 不可用时自动切 InMemoryMemory

### 6. 平台隔离
- 条件编译分离 H5 / 小程序逻辑
- AudioContext(H5) ↔ InnerAudioContext(小程序)
- 画布、录音、订阅消息各自独立实现

## 六、微信订阅推送流程

```
用户创建议程(开提醒)
  ↓
uni.requestSubscribeMessage (微信原生授权弹窗)
  ↓ 允许
POST /api/wechat/subscribe-result (上报授权)
  ↓
定时扫描 scan_and_push_reminders (每分钟)
  ↓ remind_at <= now
POST 微信 subscribe/send API
  ↓
用户收到「服务通知」
  📅 提醒事项: 🎯 项目周会
  事项主题: 项目周
  开始时间: 2026年06月02日 15:00
  事项地点: 会议室A
```

## 九、演示路径

1. 打开微信小程序 → 首页日历视图
2. 点麦克风 → 说「明天下午三点开会讨论产品方案」
3. ASR 实时显示识别文字 → Skill Router 命中创建意图
4. LLM 提取时间+标题 → 直接创建 → TTS 语音回复「已添加」
5. 创建时开启提醒 → 微信原生授权弹窗 → 允许
6. 到时间 → 收到微信服务通知推送 🎯
7. 说「明天有什么安排」→ 查询日程 → 语音播报
8. 说「删除明天的会」→ 确认 → 删除 → 日历同步刷新
